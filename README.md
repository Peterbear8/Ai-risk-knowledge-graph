## 🏗️ 分類體系：

全新設計的 AI 風險分類體系，融合多個國際框架的優點：

| 領域 | 名稱 | 涵蓋範圍 |
|------|------|----------|
| D1 | 資料與隱私風險 | 資料投毒、偏見、隱私洩露、未授權使用 |
| D2 | 模型安全風險 | 對抗攻擊、提示注入、模型竊取、幻覺 |
| D3 | 系統與基礎設施風險 | 供應鏈、來源篡改、部署環境、資源濫用 |
| D4 | 應用與互動風險 | 失控行為、過度依賴、輸出濫用、多智能體 |
| D5 | 公平與倫理風險 | 演算法歧視、透明度、人類自主性、有害內容 |
| D6 | 治理與合規風險 | 合規缺口、問責不足、社會衝擊、環境影響 |

## 🔍 互動功能

- **拖曳與縮放** — 探索圖譜結構
- **點擊節點** — 查看風險詳情（說明、嚴重度、責任角色、生命週期等）
- **關鍵字搜尋** — 中英文搜尋風險情境
- **系統類型篩選** — 按 AI 架構過濾（RAG、Agent、LLM、傳統 ML 等）
- **嚴重度篩選** — 聚焦高風險情境
- **視角切換** — 風險領域 / 🇹🇼 台灣法規原則 / 🇹🇼 風險類型

## 📐 多維度標註

每個風險情境包含以下維度：

- **Risk Source** — 風險來源（Human-Internal / Human-External / AI-System / Systemic）
- **Responsible AI Actor** — 責任角色（依 NIST AI RMF：AI Design / Development / Deployment / Operation & Monitoring / TEVV）
- **Timing** — 生命週期階段（依 ISO 5338 七階段）
- **AI System Type** — 系統架構（8 種類型）
- **Industry** — 行業別（7 種行業）
- **Severity** — 嚴重程度（1-5）

## 🇹🇼 台灣法規整合

整合台灣數發部 AI 基本法框架：
- **7 大原則**：隱私保護與資料治理、資安與安全、公平與不歧視、透明與可解釋、人類自主、問責、永續發展與福祉
- **3 大風險類型**：技術設計缺陷、部署操作與人機互動、社會結構與環境衝擊
